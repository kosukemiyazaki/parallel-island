## 2026-06-19

- **技術選定: Godot 4.x / GDScript に決定。** 理由: 完全版の最終像（本物の2Dゲームエンジン、
  パラレルワールド描画、シーン構造）に最も合う。トレードオフは「Claude Code がヘッドレスで
  動作確認しづらい」点 → 動作確認は人間が Godot エディタで再生して行う運用にする。
- **開発方式: 方式A（並列AIチーム）を採用。** 動画の「役割フォルダ」をコードプロジェクト向けに
  翻訳 → git worktree で workstream を分けて並列。まず skeleton-first で骨組みを1人で作り、
  共有インターフェースを固定してから character / sim / ui に分割する。
- **AI連携の方針: 日常行動はルールベース（AIを呼ばない）。** AIは閾値イベント・キャラ間イベント
  （約10回/ゲーム内日）と介入解釈（約3回/日）のみ。**PHASE 1 では AI 呼び出しは未実装（スタブ）**で
  進め、まず「性格による行動の差」をルールベースだけで可視化することに集中する。
- **PHASE 1 のゴールを1点に固定:** 同じ状況に置いた3〜5人が、5軸パラメータの違いだけで
  別々の行動をとる様子が俯瞰で見え、クリックでログが読める。これが見えたら成功。
