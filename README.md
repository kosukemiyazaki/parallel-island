# PARALLEL ISLAND

> 衣食住と教育が保障された島に人々を放ち、何が生まれるかを観察・介入する、神視点のシミュレーション。
> 生存競争から解放された人間は何を求めるのか——同じ性格でも、経験が違えば別の人生を歩む。その差異を眺める作品。

**▶ プレイ（ブラウザ／スマホ可）: https://kosukemiyazaki.github.io/parallel-island/**

※ 初回は wasm のダウンロードに数秒かかります。横画面推奨。

## これは何か

5人の島民が、それぞれ **5軸のパラメータ**（自己中心性／衝動性／変化適応性／自己肯定感／共感性）を持って暮らしています。
プレイヤーは「神」として島を眺め、時に「出来事」を起こして介入します。

- **性格で行動が分岐する** — 同じ状況でも、ある者は人に詰め寄り、ある者は動かず、ある者は我が道を行く。
- **人間関係が時間で育つ／壊れる** — 接触で好意・嫌悪が蓄積し、暖色／寒色の関係線で見える。暗数（突然の離反）で波乱も起きる。
- **神だけが"嘘"を見抜ける** — キャラの「行動ログ（事実）」と「自己申告ログ（本人の言い分）」はズレる。避けられているのに「みんなとうまくやれている」と言う——その乖離を覗き見るのがこの作品の核。

クリアはありません。プレイヤーが「見たいもの」を見るタイプの作品です。

## 技術

- **エンジン**: Godot 4.6 / GDScript（C# 不使用）
- **構成**: 見下ろし型2D。日常行動は **ルールベース**（5軸の重み付き抽選）で生成し、AI/LLM は使用していない。
- **公開**: HTML5 書き出し → GitHub Pages（静的ホスティング、`thread_support=false`）。
- **素材**: [Kenney "Tiny Dungeon"](https://kenney.nl/assets/tiny-dungeon)（CC0）/ フォント [Noto Sans JP](https://fonts.google.com/noto/specimen/Noto+Sans+JP)（OFL）。

## どう作ったか（開発プロセス）

このプロジェクトは **Claude Code を「並列のAIチーム」として運用する開発法**の実践として作られました。能力獲得そのものが目的です。

- **多視点の設計レビュー会議**: ゲームデザイン／UX／プロデューサー／批判役など、役割の違う複数エージェントに設計書とコードを読ませて討論させ、その結論を実装に反映（記録は [`knowledge/decisions.md`](knowledge/decisions.md)）。
- **ヘッドレス検証**: 変更のたびに Godot を `--headless` で起動し、スクリプト/実行時エラーと、行動分布・関係値（`PI_PROBE`）を自動チェック。
- **実描画の自己デバッグ**: Playwright で実ブラウザに公開ビルドを開き、スクリーンショットを撮って目視 → 文字化けやレイアウト崩れをその場で修正。
- **反復公開**: 修正 → 書き出し → GitHub Pages へ自動反映、を高速に回す。

設計思想・意思決定の全履歴は [`docs/PARALLEL_ISLAND_設計書.md`](docs/PARALLEL_ISLAND_設計書.md) と [`knowledge/decisions.md`](knowledge/decisions.md) にあります。

## 遊び方

- **キャラをタップ** … その人の5軸・人間関係・行動ログ・自己申告ログを開示。
- **「出来事を起こす」ボタン** … 島にイベントを発生。性格ごとに反応が割れる（飛びつく／無視する）。
- 画面下の **「島の出来事」** … 打ち解け・離反などの節目が流れる。
