## これは何か
衣食住と教育が保障された島に人々を放ち、何が生まれるかを観察・介入する神視点シミュレーション。
プレイヤーは「神」として眺め、時に介入し、パラレルワールドを分岐させて人間の本質を探る。

## 絶対に守る設計思想（妥協しない）
- **説明しない。見せる。** テキストで理由を語らない。動き・間・距離感で伝える。思考ログは持たない。
- **やらせ感を排除。** 全ての履歴は本当に積み上がったものだけ。演出のための嘘ログを作らない。
- **クリア概念なし。** プレイヤーが自分の見たいものを見る。ゴールを押し付けない。
- **観察の楽しさ × 介入の楽しさの二層。** 眺めるだけでも成立、仮説→実験もできる。

## 技術スタック（確定 2026-06-19）
- **Godot 4.x / GDScript**（C# は使わない）。2D、UnderTale 寄りのシンプル＋温かみ。
- シーンは `scenes/`、スクリプトは `scripts/`。`.tscn`/`.gd` はテキスト＝git で差分管理する。
- 動作確認（再生・スクショ）は **人間が Godot エディタで行う**前提。AI はコードを書く側。

## PHASE 1 の唯一のゴール
「**性格による行動の差**」が見えた瞬間にこのゲームは勝ち。それ"だけ"を証明する最小構成。
- キャラ 3〜5人、5軸パラメータ設定済み
- 同じ状況・同じイベントに置くと、全員が違う行動をとる
- 俯瞰で動きが見える
- キャラをクリックするとログが見える
- **省く**：パラレルワールド／巻き戻し／介入システム／タイムライン画面（後フェーズ）

## アーキテクチャの要（PHASE 1）
- **5軸パラメータ**：自己中心性／衝動性／変化適応性／自己肯定感／共感性。数値で管理、初期は非公開。
- **日常行動はルールベースで生成する。AI を呼ばない。** 軸ごとに「傾向」だけ定義し、掛け算で組み合わせが自然に出る設計（例：衝動性高→行動速い／自己肯定感低→他者確認が多い → 重なると「速く動くが頻繁に確認する」）。
- **AI 呼び出しは PHASE 1 では未実装でよい**（スタブ）。設計書 §4 の自律行動/介入プロンプトは後フェーズ。まずルールベースで"差"を出すことに集中。
- **ログは2種類だけ**：`行動ログ`（実際に起きたこと）と `自己申告ログ`（他者に言ったこと）。この乖離で記憶の歪みを表現。思考ログは作らない。
- シミュは**リアルタイムで動き続ける**。時間速度は 一時停止/等速/高速/超高速 を後で。

## 並列開発（方式A）のルール
1. **skeleton-first**：まず1セッションで骨組み（プロジェクト構成・5軸の型・tickループの空箱・俯瞰画面の枠・ログ表示）を作り、**共有インターフェースを固定**する。
2. 固まってから **git worktree** で workstream を分けて並列：
   - `character`：5軸パラメータ／変動／暗数／トリガー
   - `sim`：ルールベース日常行動エンジン／3レイヤー（日常/関係性/イベント）
   - `ui`：俯瞰画面・キャラクリック→ログ表示
3. **同じ `.tscn` を複数セッションで同時編集しない**（衝突源）。シーン境界で分担する。
4. 進捗監視は Claude Code デスクトップのセッション一覧 / Agent View。

## コーディング規約（GDScript・軽め）
- 命名：関数/変数 `snake_case`、クラス/ノード型 `PascalCase`。
- 可能な限り型注釈をつける（`var hp: int = 0` / `func step(dt: float) -> void:`）。
- マジックナンバーは避け、パラメータは1箇所に集約（後で図鑑/変動設計に効く）。
- 1コミット=1意味のある単位。コミットメッセージは日本語でよい。

## やってはいけない
- セリフや説明文でキャラの内面を直接語ること（設計思想違反）。
- 演出のための嘘ログ・固定シナリオの埋め込み。
- PHASE 1 で後フェーズ機能（パラレル/巻き戻し/介入/タイムライン）に手を出すこと。
